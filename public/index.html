Almost Living Context

MCP server setup for local living context.

Run the Almost Living Context service, attach the MCP stdio server, and give agents a small toolset for pulling context, adding frames, compacting old work, and checking stale claims.

Run locally Add to an MCP client Review tool list
Context service Hono API over SQLite or Postgres. Stores durable substrate items and frame history.
MCP stdio server Thin tool wrapper over the service at SERVICE_URL.
Base rollup: compact durable context
Recent frames: top unarchived deltas
Substrate: decisions, items, gotchas, components, notes

Quick Start

Start with the offline demo, then run the service and MCP server in separate terminals.

1 Install and verify

The demo uses the deterministic stub adapter, so no model or API key is required.

git clone https://github.com/Sentry-Partners/ALC.git
cd ALC
npm install
npm run demo
2 Run the service

The API defaults to SQLite at .alc.sqlite and listens on port 8787.

cp .env.example .env
npm run migrate
npm run dev
3 Run the MCP server

Keep the API running. The MCP process communicates over stdio and calls the service URL.

SERVICE_URL=http://localhost:8787 npm run mcp

Client Config

Point any MCP-compatible client at the repo checkout and run the npm MCP script.

{
  "mcpServers": {
    "almost-living-context": {
      "command": "npm",
      "args": ["run", "mcp"],
      "cwd": "/absolute/path/to/ALC",
      "env": {
        "SERVICE_URL": "http://localhost:8787"
      }
    }
  }
}
Replace /absolute/path/to/ALC with your local checkout. The MCP server expects the context service to be running before the client calls tools.

MCP Tools

These are thin wrappers over the local service API.

ctx_pull(k)

Read the base rollup plus the top k unarchived frames.

ctx_push_frame(title, body)

Append a new delta frame to the time-linear stack.

ctx_upsert_item(kind, key, title, body, status, refs, tags)

Create or update durable substrate context by stable key.

ctx_query_items(kind, status, q, key)

Search the queried substrate instead of restating durable context in prose.

ctx_compact(keep)

Archive older frames and rewrite the base rollup with the selected LLM adapter.

ctx_check_staleness(frame)

Compare one frame against current substrate items and report drift.

LLM Adapters

Use stub for offline runs, or switch to a local or hosted provider through env vars.

LLM_PROVIDER=stub

# or
LLM_PROVIDER=ollama
OLLAMA_URL=http://localhost:11434
OLLAMA_MODEL=llama3.1

# or
LLM_PROVIDER=openai
OPENAI_API_KEY=...

# or
LLM_PROVIDER=anthropic
ANTHROPIC_API_KEY=...
